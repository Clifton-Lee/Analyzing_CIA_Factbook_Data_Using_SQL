Analyzing CIA Factbook Data Using SQL
¶

Last Modified: July 1, 2021
¶

In this project, I'll be using data from the CIA World Factbook, which is a compilation of information on every nation on the planet. Aspects of the Factbook includes demographic information such as the following:

  • population: The total number of people on the planet.

  • population_growth: The yearly percentage rate of population growth.

  • area: The total area of land and water.

In this project, I will use SQL to analyze data from this database in a Jupyter Notebook and find out which countries are densely populated. If you would like to look at this database, you can download it here SQlite factbook.db database

I can see the first few rows of the facts table from the execution above. Here are some of the column descriptions:

  • name: the country's name.
  • area: the entire land area of the nation (both land and water).
  • area_land: the land area of the nation in square kilometers.
  • area_water: the water area of the nation in square kilometers.
  • population: the total number of people in the nation.
  • population_growth: the percentage of the country's population growth.
  • birth_rate: the birth rate of the nation, or the number of births per 1,000 inhabitants.
  • death_rate: the country's death rate, or the number of deaths per 1,000 inhabitants per year.

I'll now compute some summary statistics and check for any outlier nations.

In [4]:
%%sql
SELECT MIN(population) AS 'Minimum Population',
       MAX(population) AS 'Maximum Population',
       MIN(population_growth) AS 'Minimum Population Growth',
       MAX(population_growth) AS 'Maximum Population Growth' 
  FROM facts;

 * sqlite:///factbook.db
Done.
Out[4]:

0 7256490011 0.0 4.02

In the summary data shown above, I notice a few intriguing things:

  • There is a country with a population of 0
  • There is a country with a population of 7,256,490,011( or more that 7.2 billion people)

I'll use subqueries to narrow my search to only these nations.

In [5]:
%%sql
  SELECT name, population
    FROM facts
   WHERE population = (SELECT MIN(population)
                         FROM facts )
      OR population = (SELECT MAX(population)
                         FROM facts )
ORDER BY population;

 * sqlite:///factbook.db
Done.

The table above seems to include a row for the whole world, which explains the population of nearly 7.2 billion people. There also seems to be a row in the table for Antarctica, which explains the population of 0. This seems to correspond to the CIA Factbook page on Antarctica:

Now that I know this, I should recalculate the summary statistics I generated before, excluding the row for the whole world this time.

  SELECT MIN(population) AS 'Minimum Population',

Now I'm going to find out what the typical population and area look like.

In [7]:
%%sql
SELECT ROUND(AVG(population),0) AS 'Average Population',
       ROUND(AVG(area),2) AS 'Average Area'
  FROM facts;

Conclusion¶

To conclude, I'll expand on the query I made for the previous code cell to identify countries that are densely populated. I'll identify countries that have the following:

  • Above-average values for population.

  • Below-average values for area.

It's possible to write this query by copying the values I calculated on the previous screen, but it is always better to find a dynamic way to do this without hard-coding them!

In [8]:
%%sql
SELECT name, population, area
  FROM facts
 WHERE population > (SELECT AVG(population)
                       FROM facts)
   AND area < (SELECT AVG(area)
                 FROM facts);
